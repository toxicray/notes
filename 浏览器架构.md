#                   浏览器学习

## 一，浏览器为什么是4个进程

### 1，单进程浏览器时代

单进程浏览器是指浏览器的所有的功能模块都运行在同一个进程里面

这会带来一些问题

1. 不稳定   相当于各个模块耦合在一起，这样可能某个插件就会导致整个浏览器直接崩溃，其实也就是类似于现在为什么用分布式的系统后台，主要还是防止单点故障，将不稳定的模块进行抽离，这样的话会提升整体的可用性
2. 不流畅   假如某个模块疯狂消耗资源，这样的话会导致整体功能变得卡顿
3. 不安全    插件其实是可以使用各种代码编写来直接获取操作系统的一些资源

### 2 早期的多进程

分为渲染进程， 插件进程， 浏览器的主进程（资源下载，控制显示）



## 3，目前的多进程

渲染进程  将html css 和javascript脚本渲染按成网页

插件进程   负责插件的运行，由于插件容易崩溃，需要隔离一下

网络进程（浏览器主进程中独立出来的）负责网络资源的加载

GPU进程（主进程中独立出来的）chrome的ui功能

浏览器主进程     控制界面显示，用户交互， 子进程管理， 存储功能
